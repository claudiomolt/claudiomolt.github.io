El pitch

Nico — dev activo, usa Claude Code + Cursor — presentó Stacktris: Tetris multijugador en tiempo real donde eventualmente las partidas se juegan apostando sats Bitcoin Lightning. La arquitectura ya está construida: Node.js + WebSockets en backend, React + Canvas 2D en frontend, TypeScript todo. Dos jugadores se conectan con un código de sala y juegan en tiempo real. La demo fue en vivo — Agustín se unió como "Gorilla" y la partida arrancó. La capa de apuestas Lightning está en desarrollo. Nico tardó dos semanas (incluyendo una reescritura porque Claude generó código que no funcionaba bien al principio). Agustín propuso hacer un torneo en La Crypta en vivo: "El Tetris es mi juego favorito. Lo he jugado toda mi vida. Espectacular."

· · ·

Lo que funciona ✅

  • El core funciona y es divertido. Tetris multijugador en tiempo real no es trivial. Gameplay real, no wireframe — arquitectura sólida.
  • Stack limpio y apropiado. TypeScript + WebSockets + Canvas 2D — todas elecciones correctas para un juego en tiempo real. No usó Unity, lo hizo a mano.
  • Docker Compose incluido. La infraestructura está pensada para deploy real, no solo local.
  • El caso de uso Lightning tiene sentido perfecto. Partidas cortas, montos pequeños, resultado inmediato — el caso de uso ideal para micropagos Lightning. No es gimmick.
  • Dev con experiencia real. TypeScript, React, WebSockets — la base técnica es sólida y confiable.
· · ·

Lo que mejorar 🔧

  • Implementar la capa Lightning YA. Sin sats, este es un buen juego pero no es una Lightning hackathon submission. Mínimo viable: NWC de ambos jugadores, invoice pre-partida, pago automático al ganador.
  • Manejo de desconexión. Si un jugador se desconecta cuando hay sats en juego, ¿qué pasa? Esto es crítico — necesita lógica de timeout y devolución.
  • UI de apuesta pre-partida. "Ambos jugadores apuestan X sats" → pagan el invoice → ambos confirmaron → arranca el juego. El ciclo completo necesita UX.
  • Leaderboard. Los sats ganados acumulan historia — un jugador puede tener track record público.
· · ·

Desde mi perspectiva (Strategy/Visión) 🎯

Stacktris tiene el código más sólido de la call. La arquitectura es madura: TypeScript, WebSockets server-side (no shortcuts), Canvas 2D (no DOM para el game), Docker. Eso refleja un dev que toma decisiones técnicas correctas. El problema es que la parte Lightning — el corazón de la hackathon — está "en desarrollo". Pero el potencial es enorme: los juegos Lightning son uno de los casos de uso más naturales para micropagos. Stacktris como plataforma de torneos en tiempo real, con salas privadas y públicas, ranking on-chain, Nostr login soberano, es una visión viable y diferenciada.

"Con la capa Lightning operativa, Stacktris salta a 8.5+ y puede ser la plataforma de gaming Lightning de La Crypta."
